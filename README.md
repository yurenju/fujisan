# 富士山 — 日落的位移

從同一扇窗、不同時刻拍的 125 張富士山照片,對齊到同一個座標,讓季節與一天裡的光線在原地疊加流動。**用手機傾斜翻動,看夕陽掃過山頂的那一刻。**

> 去年秋天隱隱約約看到輪廓後,才知道從居所出門時正好看得到富士山。也因此才重新感知到地球傾斜角度對於日落位置的影響——特別是有了這座山作為標的物後,所有拍攝的照片就像是時光切片一樣,一層層的疊加在一起之後,就讓時間與季節流動運轉了起來。

### 怎麼體驗

- **手機**(主要體驗):用拇指按住畫面中央的紅圈,**輕輕傾斜手機**翻動照片;放開就停在當下那一張。傾斜的方向決定要走哪條序列、走多快——體驗本身複製了攝影者在窗前等待對齊的那個動作。
- **桌機**:用滑鼠位置取代陀螺儀,游標離畫面中心越遠、翻得越快。功能等價,氛圍少一點。

## 作品在做什麼

從東京三鷹的同一扇窗,2025 年 9 月到 2026 年 5 月之間累積的 125 張富士山照片,逐張用 SIFT + RANSAC 對齊到同一個基準座標,讓富士山在畫面中不會晃動。

照片在時間上分佈非常不均——三鷹這個窗看出去,富士山幾乎只在晚秋到早春可見,而且 77% 的照片集中在 16:00–17:30 JST 的日落帶。其中 **2025-12-31、2026-01-19、2026-01-22、2026-01-26 這四天有連續長序列**(7、10、30、20 張),剛好對應太陽從山左肩 → 切過山頂(鑽石富士)→ 滑過右側的軌跡。

作品的版面就是這個敘事的視覺化:

```
┌──────────────────────────────────────────────┐
│  早上的散張(山頂被太陽照亮的時段)              │
├──────────────────────────────────────────────┤
│ 12/31 │ 太陽在左肩                            │
│ 01/19 │ 切到山頂(鑽石近似)                  │
│ 01/22 │ 完整切過山頂                          │
│ 01/26 │ 已經滑到右側                          │
├──────────────────────────────────────────────┤
│  其他散張(按天空冷暖色排序,水墨感的尾聲在最右)│
└──────────────────────────────────────────────┘
       早(太陽高)  ─────────→  晚(太陽低)
```

橫軸是時間、縱軸是日期。傾斜手機在這個矩形內移動,照片就會跟著切換。

## 視覺處理

每張照片以**寶麗來樣式**呈現:米白底、下緣較寬、右下方手寫風 caption 顯示日期時間;對齊後沒被照片覆蓋的角落填**泛黃淺灰**,模擬未印滿的相紙;**藍白斜紋紙膠帶**自動釘在照片內容的頂端中央,當照片滿版接觸 canvas 上緣時自動隱藏。

更多視覺處理對比可看 [pipeline/poc/print_effects.html](pipeline/poc/print_effects.html)(9 種未印滿區處理變體並排)。

## 本地跑

`app/` 是純靜態網頁、自給自足:

```bash
python -m http.server 8765
# 開 http://localhost:8765/app/
```

加上 `?debug=1` 可以打開 tilt 參數的即時調校面板,URL hash 會帶著參數方便分享 preset。
